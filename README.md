# 010-Uber-Data-Analysis-SQL-Project

🚖 Uber Ride Booking SQL Data Analysis Project
📌 Project Overview

This project demonstrates SQL-based exploratory data analysis (EDA) on an Uber ride booking dataset. The objective is to answer real-world business questions that can help ride-sharing companies improve operations, maximize revenue, reduce cancellations, enhance customer satisfaction, and optimize driver performance.

The project contains 24 business-driven SQL queries covering revenue analysis, ride demand, customer behavior, cancellation analysis, driver performance, and operational insights.

📈 Revenue Analysis
1. Total booking revenue by month
Calculates monthly revenue generated from ride bookings.
2. Vehicle types generating the highest revenue
Determines which vehicle categories contribute the most revenue.
3. Peak booking hours
Identifies the busiest hours of the day.
4. Highest ride demand by day
Shows which days receive the most bookings.
5. Average booking value
Calculates the average revenue earned per booking.
❌ Cancellation Analysis
6. Overall cancellation rate
Measures the percentage of cancelled rides.
7. Pickup locations with the highest cancellations
Identifies locations experiencing frequent cancellations.
8. Top customer cancellation reasons
Analyzes why customers cancel rides.
9. Top driver cancellation reasons
Examines the most common reasons drivers cancel bookings.
10. Vehicle types with the most cancellations
Determines which vehicle categories have higher cancellation rates.
⭐ Driver Performance Analysis
11. Vehicle types with the highest driver ratings
Evaluates average driver ratings by vehicle type.
12. Impact of driver ratings on booking completion
Analyzes whether higher-rated drivers complete more rides.
13. Pickup locations with the best-rated drivers
Identifies locations with consistently higher driver ratings.
14. Average ride distance by vehicle type
Measures average trip distance for each vehicle category.
👥 Customer Behavior Analysis
15. Most frequently used payment methods
Identifies customer payment preferences.
16. Percentage of incomplete rides
Calculates how many bookings remain incomplete.
17. Main reasons for incomplete rides
Determines the leading causes of incomplete trips.
18. Pickup and drop locations with the highest incomplete rides
Locates operational problem areas.
19. Hours with the highest incomplete rides
Shows when incomplete rides occur most frequently.
📍 Operational Analysis
20. Pickup locations generating the highest revenue
Identifies the most profitable pickup areas.
21. Most frequently traveled routes
Shows the most common ride origins.
22. Areas with the longest average ride distances
Analyzes long-distance travel patterns.
23. Booking trends over time
Tracks booking volume by month and year.
24. Cancellation trends over time
Measures how cancellation rates change across different months.

⭐ Key Takeaways

This project highlights how SQL can transform raw ride-booking data into actionable business insights. By answering practical business questions, it demonstrates proficiency in data analysis, KPI reporting, and decision support—skills that are highly relevant for Data Analyst, Business Intelligence, and SQL Developer roles.
